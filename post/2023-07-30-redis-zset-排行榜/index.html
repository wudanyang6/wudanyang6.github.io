数据量级较大的实时排行榜

一、问题#

数据量级上亿，如何在客户端实时显示一个天级实时排行榜？

数据格式为：

key:value

根据 value 的值显示 key 的排行榜，如：

id1  =>  10
id2  =>  9
id3  =>  8
id4  =>  7
id5  =>  6
...

要求这个排行榜随着用户操作 value 变化时，可以做到实时变化，但是排行榜可以只显示前10000条

二、解决方案#

我们都知道 redis 的 zset 方法，可以将数据串成一个跳表，可以获取数据的排行榜数据

如下面在 redis-cli 中的操作：

# 在redis中添加一个元素board，类型为zset，key：id1，value：10
ZADD board 10 id1
ZADD board 9 id2
ZADD board 8 id3
... 继续添加

# 取出前1万条
ZREVRANGE board 0 10000

2.1 暴力存储#

我们可以利用上面的 zset 将一亿条数据全都存起来，然后每次 ZREVRANGE board 即可获取排行榜前一万名

这样的优点是：

  1. 程序逻辑简单，维护简单，只需要写两个函数，一个是修改数据的函数，一个是获取排行榜的函数

如果数据量级比较小的话，这样做是没问题的，但是如果量级上亿的话，就会出现几个缺点：

  1. redis 负载高：修改数据的函数每天会被调用上亿次，会造成热点key问题
  2. 存储成本高：redis 的本质是个数据库，存储上亿条数据会使用巨量的内存，需要花很多钱才能搞定，成本也上去了

2.2 对超过一定量级的数据进行存储#

我这里给出的解决方案是：

先对历史数据进行统计，找到一个合理的 `阈值`，只有大于阈值的数据才能进 zset
不仅 redis 的负载可以降下来，存储的成本也没这么高了

这里面有几个步骤，下面进行详细说明

2.2.1 数据统计#

统计数据建议使用离线统计的方式，将过去几天的天级数据分段统计出来

select 
	count(distinct if(value >= 10, qid, null)) as g_10,
	count(distinct if(value >= 20, qid, null)) as g_20,
	count(distinct if(value >= 30, qid, null)) as g_30,
	count(distinct if(value >= 50, qid, null)) as g_50,
	count(distinct if(value >= 100, qid, null)) as g_100,
	count(distinct if(value >= 1000, qid, null)) as g_1000,
	count(distinct if(value >= 10000, qid, null)) as g_10000
from 
table_of_value
where dt='{date}'
and value >= 10;

你必须给你的阈值留有一个很大的buffer空间，而且如果value的值会变小的话，
最好是能每天根据前一天的数据更新一个阈值

2.2.3 阈值准入函数#

func ChangeValue(key string, value int) error {
	// 必须要记录每次的值，但是不进行排序
	newValue = incrValueToRedis(key, value)
	...
	updateCache(key, newValue)
	...
	return nil
}

func updateCache(key string, value int) nil {
	if value < THRESHOLD {
		return nil
	}
	
	redis.ZADD("board", value, key)
}

2.2.4 取排行榜#

func GetBoardTopN(N int) error {
	if N > 10000 {
		return error.New("not support")
	}

	// 取出数据并映射成 struct
	zsetSlice := redis.ZREVRANGE("board", 0, N)

	ret := make([]string)
	for _, v := range zsetSlice {
		if v.Value == THRESHOLD {
			ret = append(ret, v)
		}
	}

	return ret
}

三、总结#

这种方法一定要注意，在存储量级和修改量级可控的情况下，阈值要选好，最好加上监控报警以备不时之需
